Understanding Submodular Reinforcement Learning

10 min read

Standard reinforcement learning assumes that rewards are additive over time. But many real-world problems exhibit diminishing returns: the value of collecting new information decreases as you accumulate more, diverse sensor placements provide more coverage than clustered ones, and exploring new states becomes less valuable when you've already seen similar ones.

This is where submodular reinforcement learning comes in. In this post, I'll explain what submodularity means, why it matters for sequential decision-making, and the key challenges in developing algorithms with provable guarantees.

What is Submodularity?

A set function f is submodular if it exhibits the diminishing returns property. Formally, for any sets A, B with A contained in B, and any element x not in B:

f(A + x) - f(A) ≥ f(B + x) - f(B)

In words: adding an element to a smaller set provides at least as much marginal value as adding it to a larger set. This captures the intuition that "the first few observations are more valuable than later ones."

Examples in Practice

Submodular objectives appear in numerous applications:

  • Sensor placement: Maximizing information coverage while placing sensors
  • Experiment design: Selecting informative experiments for scientific discovery
  • Informative path planning: Routing robots to maximize area coverage or data collection
  • Feature selection: Choosing diverse features for machine learning models

The Challenge: Sequential Decisions

Traditional submodular optimization assumes you make all decisions at once. But many real problems are sequential: you take an action, observe an outcome, then decide what to do next. This creates a fundamental tension.

Consider a robot exploring an environment. At each step, it must decide where to move based on what it has observed so far. The total information gained is a submodular function of all visited locations, but the robot must commit to each location sequentially without knowing what it will find. 

# The key insight: actions have two effects
def take_action(state, action):
    # 1. Immediate effect on the environment
    next_state = transition(state, action)

    # 2. Contribution to the accumulated set
    #    (affects future marginal values)
    visited_locations.add(action)

    # The submodular reward depends on ALL
    # visited locations, not just the current one
    return next_state

Why Standard RL Fails

Standard RL algorithms assume rewards decompose additively: the total return is the sum of individual rewards. This enables powerful techniques like dynamic programming and temporal difference learning.

But with submodular objectives, the reward at time t depends on all previous actions, breaking the Markov assumption. The value of visiting a location depends not just on the current state, but on the entire history of visited locations.

Key Technical Challenges

  • Non-Markovian rewards: The reward depends on the full trajectory, not just the current state-action pair
  • Computational hardness: Even with known dynamics, the problem is NP-Hard to solve optimally
  • Exploration-exploitation: How do you explore efficiently when rewards are non-additive?

Our Approach: Approximate Optimality

Since optimal solutions are computationally intractable, we focus on algorithms with approximation guarantees. The key insight is to leverage the structure of submodular functions to design greedy-like algorithms that provably achieve a constant fraction of the optimal value.

The classical result for submodular maximization shows that the greedy algorithm achieves a (1-1/e) approximation. Our work extends this to the sequential setting, developing algorithms that maintain similar guarantees while handling:

  • Unknown transition dynamics (learning from interaction)
  • Continuous state and action spaces
  • Safety constraints on the exploration process

Looking Forward

Submodular RL opens up new possibilities for sequential decision-making with complex objectives. Beyond the applications I've mentioned, it connects to broader questions about how to design AI systems that can reason about diminishing returns, diversity, and coverage.

In future posts, I'll dive deeper into the technical details of our algorithms and discuss connections to related problems like safe exploration and multi-agent coordination.
